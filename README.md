### ⏱ 1부 – 아이스브레이킹 & 방향 공유 (20분)

- 간단 자기소개
    - 이름
    - 알고리즘 경험 (한 줄)
    - IM 목표 여부

---

### ⏱ 2부 – 가벼운 발표 (30분)

- 미리 풀어오라고 **강요 ❌**
- 대신:
    - “최근 풀었던 백준 문제”
    - or “이해 안 됐던 과제/실습문제”

👉 **맞은 문제 / 틀린 문제 둘 다 OK**

발표 포맷:

1. 문제 설명 (1분)
2. 접근 방식 (2분)
3. 헷갈렸던 부분 (1분)

---

### ⏱ 3부 – 실전 풀이 (30분)

- 다 같이 **IM 기출** **문제 1개**
    - 예:
        - 10871 X보다 작은 수
        - 2562 최댓값

진행 방식:

- 5분: 문제 읽고 접근 아이디어 말로 정리
- 10분: 각자 구현
- 10분: 풀이 비교

👉 **“문제 이렇게 읽는다” 감각 심기**

---

### ✅ 인원 구조 재정의

저희 스터디 인원이 총 10명이기 때문에

👉 **5명 ×2그룹** 씩 나눠서 진행합니다.

(오프라인 모임은 **같이**, 발표는 **그룹 단위**)

- A그룹 / B그룹
- 각 그룹에서 매주 **발표자 1명씩 돌아가며 진행**

이렇게 하는 이유

- 발표 부담 ↓
- 듣기 집중도 ↑
- 빠지는 사람 생겨도 스터디 유지
